# Human Feedback Materials

This folder holds materials for cold-reader testing and product feedback.

Its purpose is to evaluate whether user-facing artifacts from the framework are:
- understandable
- credible
- actionable
- clearly differentiated (for example, Promoted vs Watchlist)

## Current pack

The current feedback pack is:

- `packs/remote_workflow_pack_v1/`

It should be used with the current Remote Workflow Opportunity Brief artifacts.

## How to use this folder

1. Share one small feedback pack with a cold reader.
2. Ask the standard questions in `feedback_questions.md`.
3. Record the session using `sessions/TEMPLATE.md`.
4. Summarize repeated findings in `synthesis/feedback_summary_v1.md`.

## Ground rules

- Treat this as product feedback, not backend critique.
- Focus on comprehension, usefulness, trust, and actionability.
- Do not change backend logic unless repeated human feedback points to a structural issue.
